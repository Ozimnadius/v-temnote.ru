# Документация Для Агентов

## Назначение

Эта папка содержит специализированные правила для агентов и субагентов проекта.

`AGENTS.md` остается единой точкой входа. Этот файл является индексом частных спецификаций и помогает выбрать нужный документ под задачу.

## Порядок Работы

1. Прочитать `AGENTS.md`.
2. Прочитать этот индекс.
3. Выбрать нужную спецификацию по типу задачи.
4. Действовать по выбранной спецификации.

## Спецификации

- `html-structure.md` — правила HTML-структуры, BEM, контейнеров, семантических тегов и запретов.
- `scss-structure.md` — правила создания SCSS-файлов BEM-блоков, подключения через `@forward`, медиа-запросов и hover-состояний.
- `figma-to-project.md` — правила переноса Figma-дизайна в проект с двумя ветками работы: общий скелет страницы или страница с внутренними блоками.
- `aside.md` — архитектура боковой колонки страниц: свойства `show_aside` / `show_aside_detail`, файлы `sect_aside.php`, библиотека карточек `include/aside/`.
- `layout-stages.md` — разделение работ между статической Vite-версткой и Bitrix-натяжкой.
- `block-types/README.md` — индекс правил для повторяющихся типов блоков, например слайдеров.
- `bitrix-wizard.md` — архитектура, файловая структура, навигация шагов, константы, services и подводные камни системы мастеров установки Bitrix.
- `webcomp-shop-wizard-plan.md` — план работ для нового мастера установки решения `webcomp:shop`: шаги, services, шаблон, sale-настройки и импорт мегаменю.

## Добавление Новых Правил

Новые агентские правила добавляются только в `docs/layout-developer`.

Если появляется новая спецификация:

- создать отдельный MD-файл с коротким смысловым названием;
- добавить ссылку на него в этот индекс;
- не дублировать общие правила согласования из `AGENTS.md`;
- не создавать `PLANNING.md`, `docs/plans` или отдельные планы страниц и блоков.
