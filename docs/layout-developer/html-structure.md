# HTML-структура

## Назначение

Этот документ описывает базовую HTML-структуру проекта и правила, которыми должны руководствоваться агенты при создании новых шаблонов, секций и компонентов.

Цель документа — сохранять единый каркас страниц, предсказуемую вложенность, BEM-именование и совместимость с Bitrix-шаблоном проекта.

## Базовый Каркас Страницы

Страница строится вокруг общего layout-контейнера `wrapper`.

```html
<body>
  <div class="wrapper">
    <div class="wrapper__panel">
      <!-- Bitrix panel -->
    </div>

    <div class="wrapper__header">
      <!-- site header -->
    </div>

    <div class="wrapper__content">
      <!-- page content -->
    </div>

    <div class="wrapper__footer">
      <!-- site footer -->
    </div>
  </div>
</body>
```

Назначение зон:

- `wrapper` — главный layout-контейнер страницы.
- `wrapper__panel` — место для панели Bitrix.
- `wrapper__header` — зона шапки сайта.
- `wrapper__content` — основное содержимое страницы.
- `wrapper__footer` — зона подвала сайта.

Агентам нельзя менять эту верхнеуровневую структуру без отдельного согласования.

## Контейнеры

Для ограничения ширины контента используется существующий класс `container`.

```html
<div class="container">
  ...
</div>
```

Правила:

- Использовать `.container` для стандартного ограничения ширины.
- Не создавать аналоги вроде `page-container`, `content-container`, `main-container`, если задача решается существующим `.container`.
- Не добавлять `.container` внутрь другого `.container` без явной причины.
- Если блоку нужна внутренняя раскладка, создавать элемент блока, например `block-name__container`.

Пример:

```html
<div class="catalog-section">
  <div class="container">
    <div class="catalog-section__container">
      ...
    </div>
  </div>
</div>
```

Для внутренних блоков страницы `.container` ставится сразу внутри зоны страницы.

Порядок вложенности:

1. Элемент верхнего блока страницы, например `main__hero`.
2. `.container`.
3. Самостоятельный BEM-блок, например `main-hero`.
4. Внутренние элементы блока, например `main-hero__container`.

Правильно:

```html
<div class="main__hero">
  <div class="container">
    <section class="main-hero">
      <div class="main-hero__container">
        ...
      </div>
    </section>
  </div>
</div>
```

Неправильно:

```html
<div class="main__hero">
  <section class="main-hero">
    <div class="container">
      <div class="main-hero__container">
        ...
      </div>
    </div>
  </section>
</div>
```

Если дизайн показывает ограниченную ширину контента, `.container` обязателен. Full-width структура допустима только если Figma явно показывает блок на всю ширину.

## BEM И Вложенность

HTML проекта строится в BEM-стиле.

Блок:

```html
<div class="product-card">
  ...
</div>
```

Элементы блока:

```html
<div class="product-card">
  <div class="product-card__image"></div>
  <div class="product-card__body"></div>
  <div class="product-card__title"></div>
</div>
```

Модификаторы:

```html
<a class="header-nav__link header-nav__link--active" href="#">
  Активный пункт
</a>
```

Правила:

- Элемент `block__element` используется только внутри своего блока `block`.
- Модификатор `block--modifier` или `block__element--modifier` не должен существовать без базового класса.
- Новый самостоятельный смысловой фрагмент оформляется как новый BEM-блок.
- Не смешивать элементы разных блоков в одной структуре без необходимости.
- BEM-названия должны быть короткими и смысловыми; длинные составные имена использовать только если без них теряется смысл.

Неправильно:

```html
<div class="product-card">
  <div class="header__user-menu">
    ...
  </div>
</div>
```

Правильно:

```html
<div class="product-card">
  <div class="product-card__actions">
    ...
  </div>
</div>
```

## Новые Блоки И Секции

HTML-тег выбирается по семантике, а не по шаблону.

Использовать:

- `section` — тематическая секция страницы с самостоятельным смыслом.
- `nav` — навигационный блок.
- `header` — шапка страницы или крупного самостоятельного блока.
- `footer` — подвал страницы или крупного самостоятельного блока.
- `main` — основное содержимое страницы, если оно не задается Bitrix-шаблоном.
- `aside` — боковая или вспомогательная область.
- `article` — самостоятельная карточка материала, статьи, новости или записи.
- `div` — нейтральная обертка, если семантический тег не подходит.

Не использовать `<section>` автоматически для любого нового блока.

Базовый принцип:

1. Выбрать HTML-тег по смыслу.
2. Назначить внешнему узлу BEM-блок.
3. Добавить `.container`, если нужно ограничение ширины.
4. Добавить внутренний элемент блока для раскладки, если он нужен.

Пример самостоятельной секции:

```html
<section class="products-section">
  <div class="container">
    <div class="products-section__container">
      ...
    </div>
  </div>
</section>
```

Пример навигационного блока:

```html
<nav class="catalog-nav" aria-label="Каталог">
  <div class="container">
    <ul class="catalog-nav__list">
      ...
    </ul>
  </div>
</nav>
```

Пример нейтрального компонента:

```html
<div class="product-card">
  <div class="product-card__image">
    ...
  </div>
  <div class="product-card__body">
    ...
  </div>
</div>
```

## Изображения

Контентные изображения можно вставлять двумя способами:

1. `picture` — если нужны разные изображения под разные разрешения.
2. `img` — если используется одно изображение без адаптивных вариантов.

Если задача запускается через skill `layout-developer` / `Верстальщик` и в блоке есть изображения, агент должен сначала уточнить способ вставки изображений. Нельзя молча выбирать `picture` или `img`, если пользователь еще не выбрал режим.

### Вариант `picture`

Использовать, когда для desktop, tablet, mobile или других разрешений есть отдельные Figma nodes, assets или явно разные версии изображения.

```html
<picture class="product-card__picture">
  <source media="(max-width: 767px)" srcset="path/to/image-mobile.webp">
  <source media="(max-width: 1199px)" srcset="path/to/image-tablet.webp">
  <img class="product-card__image" src="path/to/image-desktop.webp" alt="Описание изображения">
</picture>
```

Правила:

- BEM-класс ставится на `picture`, если для обертки изображения нужен отдельный элемент блока.
- У fallback-изображения `img` всегда должен быть BEM-класс.
- `source` используется для альтернативных разрешений или форматов.
- `alt` должен описывать смысл изображения; для декоративного изображения допустим пустой `alt=""`.
- Если ссылки на адаптивные Figma nodes не предоставлены, не придумывать дополнительные изображения.

### Вариант `img`

Использовать, когда дизайн или задача предполагают одно изображение без отдельных адаптивных версий.

```html
<img class="product-card__image" src="path/to/image.webp" alt="Описание изображения">
```

Правила:

- У `img` всегда должен быть BEM-класс.
- `alt` должен быть осмысленным или пустым для декоративного изображения.
- Не заменять `img` на `picture` без согласования выбранного режима.

## Header Как Эталон Структуры

Шапка сайта является примером текущего подхода к HTML-структуре.

```html
<header class="header">
  <div class="container">
    <div class="header__container">
      <div class="header__top">
        ...
      </div>

      <div class="header__main">
        ...
      </div>

      <div class="header__categories">
        ...
      </div>
    </div>
  </div>
</header>
```

Основные зоны:

- `header__top` — город, верхнее меню, телефоны, кнопка звонка, переключатель темы.
- `header__main` — логотип, кнопка каталога, поиск, пользовательское меню.
- `header__categories` — строка категорий.

Вложенные самостоятельные блоки:

- `header-nav` — верхнее меню.
- `phones` — телефон и выпадающий список телефонов.
- `theme-toggle` — переключатель светлой и темной темы.
- `logo` — логотип.
- `header-categories` — строка категорий.
- `header__user-menu` — меню пользователя: вход, сравнение, избранное, корзина.

При доработке header нужно сохранять разделение ответственности между этими блоками.

## Общие Запреты

Запрещается:

- Менять базовый каркас `wrapper` без отдельного согласования.
- Использовать элементы блока вне родительского блока, например `header__...` вне `.header`.
- Создавать новые аналоги `.container` без необходимости.
- Вкладывать `.container` в `.container`.
- Добавлять лишние обертки, если они не нужны для семантики, сетки, состояния или JS.
- Использовать inline-стили в HTML.
- Использовать классы компонента для другой семантики.
- Смешивать BEM-элементы разных блоков в одной структуре.
- Использовать временную демо-разметку `.demo` как основу для целевых страниц.
- Называть блоки по внешнему виду вместо смысла, например `blue-block`, `big-card`, `left-section`.
- Создавать излишне длинные BEM-названия, если есть короткий понятный вариант.

Предпочитать смысловые названия:

```html
<section class="services-section"></section>
<div class="product-card"></div>
<nav class="catalog-nav"></nav>
<div class="main-showcase"></div>
```

Избегать названий по внешнему виду:

```html
<section class="gray-section"></section>
<div class="large-box"></div>
<nav class="top-links"></nav>
<div class="main-category-showcase"></div>
```

## Минимальные Требования К Новому HTML

Перед добавлением новой HTML-структуры агент должен проверить:

- Подходит ли выбранный HTML-тег по семантике.
- Не нарушается ли базовый каркас `wrapper`.
- Используется ли существующий `.container`, если нужно ограничение ширины.
- Для внутреннего блока страницы соблюдается порядок `page__zone > .container > block`.
- Является ли новый фрагмент самостоятельным BEM-блоком или элементом существующего блока.
- Если в блоке есть изображения, выбран ли режим вставки `picture` или `img`.
- Нет ли лишних оберток.
- Нет ли inline-стилей.
- Не используются ли временные демо-классы.
